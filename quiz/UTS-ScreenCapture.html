SOAL 1 — Instalasi Java dan Encapsulation (20 poin)

Soal ini meminta Anda membuktikan Java sudah terinstall, lalu membuat class pertama dengan data terlindungi (private).

1a. Cek Java (5 poin)

Buka terminal lalu ketik:

java -version
javac -version
Screenshot: terminal yang menampilkan versi Java.

1b. Buat Class Utama (15 poin)

Buat file Java dengan nama sesuai kolom Barang dari Tabel A (contoh: Buku.java). Class ini harus punya:

  • 4 field private: nama (String), harga (double), stok (int), nim (String)
  • Constructor untuk mengisi semua field
  • Getter untuk semua field
  • Setter setHarga() dan setStok() dengan validasi:
    • Jika harga ≤ 0 → cetak [NIM] ERROR: Harga tidak valid! dan jangan ubah nilai
    • Jika stok < 0 → cetak [NIM] ERROR: Stok tidak valid! dan jangan ubah nilai
  • Method info() — mencetak semua data barang
  • Method jual(int jumlah):
    • Stok cukup → kurangi stok, cetak [NIM] Terjual [jumlah] [Barang]. Sisa: [stok]
    • Stok tidak cukup → cetak [NIM] GAGAL: Stok tidak cukup!

Isi harga dan stok sesuai Tabel B. Isi nim dengan NIM Anda.

Buat juga main() yang menjalankan: info(), jual() sukses, jual() gagal, setHarga(-1), setStok(-1).

SOAL 2 — Inheritance (20 poin)

Buat 3 subclass dari class Soal 1. Tujuannya: subclass mewarisi sifat superclass dan bisa menambah / mengubah perilaku.

Buat 3 class baru sesuai kolom Jenis A, B, C dari Tabel A. Ketiga class meng-extend class Soal 1.

Contoh untuk NIM akhiran 23 (Toko Buku):

Buku            <-- superclass (dari Soal 1)
├── Novel       <-- subclass A
├── Komik       <-- subclass B
└── TextBook    <-- subclass C

Setiap subclass harus:

  1. Pakai extends ke class Soal 1.
  2. Punya constructor yang memanggil super(...).
  3. Punya 1 field tambahan: Jenis A → String kategori  |  Jenis B → double berat  |  Jenis C → boolean premium
  4. Override info() → panggil super.info() lalu cetak field tambahan.
  5. Override jual(int jumlah) dengan cetakan tambahan:
    • Jenis A → [NIM] [JenisA]: proses standar
    • Jenis B → [NIM] [JenisB]: perlu penanganan khusus
    • Jenis C → jika premium true: [NIM] [JenisC]: PREMIUM, diskon 10%

Tambahkan di setiap constructor (superclass dan subclass):

System.out.println("[NIM] Constructor [NamaClass] dipanggil");

Buat main() yang membuat ketiga objek, panggil info() dan jual(1) masing-masing.

SOAL 3 — Polymorphism (20 poin)

Tunjukkan 2 jenis polymorphism: overriding (method sama, perilaku beda) dan overloading (nama sama, parameter beda).

3a. Overriding + Dynamic Dispatch (10 poin)

Buat array bertipe superclass yang diisi ketiga subclass:

Barang[] daftar = {
    new JenisA(...),
    new JenisB(...),
    new JenisC(...)
};

(Ganti nama sesuai tabel NIM Anda.)

Loop array, untuk setiap elemen cetak:

System.out.println("[NIM] Tipe: " + item.getClass().getSimpleName());
item.info();
item.jual(2);
System.out.println("---");

Ini membuktikan: tipe variabel sama (superclass), tapi method yang jalan milik masing-masing subclass.

3b. Overloading (10 poin)

Tambahkan 3 method hitungTotal di superclass (nama sama, parameter beda):

double hitungTotal(int jumlah) {
    // harga * jumlah
}

double hitungTotal(int jumlah, double diskonPersen) {
    // harga * jumlah * (1 - diskonPersen/100)
}

double hitungTotal(int jumlah, String kodeVoucher) {
    // kalau kodeVoucher == NIM Anda --> diskon 50%
    // kalau beda --> tidak ada diskon
}

Di main(), panggil ketiga versi dan cetak hasilnya. Gunakan angka Diskon Member dari Tabel B untuk parameter diskonPersen.

SOAL 4 — Abstraction (20 poin)

Ubah superclass menjadi abstract class dan buat interface. Tujuannya: memaksa subclass mengisi method tertentu.

4a. Abstract Class (10 poin)

Ubah class Soal 1 menjadi abstract class, tambahkan 2 method abstract:

abstract String getDeskripsi();
abstract double hitungPajak();

Setiap subclass wajib mengisi:

  • getDeskripsi() → return deskripsi yang beda per subclass.
  • hitungPajak(): Jenis A → harga × 11/100 (PPN)  |  Jenis B → harga × 5/100  |  Jenis C → 0 (bebas pajak)

4b. Interface (10 poin)

Buat interface Diskonable:

interface Diskonable {
    double hitungDiskon();
    String infoDiskon();
}

Aturan:

  • Jenis B implements Diskonable → hitungDiskon() return harga × diskonMember / 100, infoDiskon() return "Member [X]% - NIM: [NIM]"
  • Jenis C implements Diskonable → hitungDiskon() return harga × diskonPromo / 100, infoDiskon() return "Promo [X]% - NIM: [NIM]"

(Gunakan angka dari Tabel B.)

Buat main() yang membuat ketiga objek, panggil info(), getDeskripsi(), hitungPajak(); untuk Jenis B dan C panggil juga hitungDiskon() dan infoDiskon().

SOAL 5 — Strategy Pattern (20 poin)

Buat sistem kasir yang bisa ganti-ganti cara hitung diskon tanpa ubah kode kasir (Strategy Pattern).

Buat interface:

interface StrategiDiskon {
    double hitungHargaAkhir(double harga, int jumlah);
    String namaStrategi();
}

Buat 3 class yang implements interface di atas:

  1. TanpaDiskon — harga akhir = harga × jumlah (tanpa potongan).
  2. DiskonMember — harga akhir = harga × jumlah × (1 − diskonMember/100). Pakai angka dari Tabel B.
  3. DiskonPromo — harga akhir = harga × jumlah × (1 − diskonPromo/100). Pakai angka dari Tabel B.

Buat class Kasir:

class Kasir {
    private StrategiDiskon strategi;

    void setStrategi(StrategiDiskon s) {
        this.strategi = s;
    }

    void bayar(String namaBarang, double harga,
               int jumlah, String nim) {
        double total = strategi.hitungHargaAkhir(harga, jumlah);
        double diskon = (harga * jumlah) - total;
        System.out.println("=== STRUK " + nim + " ===");
        System.out.println("Strategi : " + strategi.namaStrategi());
        System.out.println("Barang   : " + namaBarang);
        System.out.println("Harga    : " + (int)harga);
        System.out.println("Jumlah   : " + jumlah);
        System.out.println("Diskon   : " + (int)diskon);
        System.out.println("Total    : " + (int)total);
        System.out.println("==================");
    }
}

Di main(), buat 1 objek Kasir, jalankan 3 transaksi dengan strategi berbeda:

  • Transaksi 1 — TanpaDiskon, beli 3 Jenis A
  • Transaksi 2 — DiskonMember, beli 2 Jenis B
  • Transaksi 3 — DiskonPromo, beli 5 Jenis C

Gunakan nama barang dan harga dari tabel NIM Anda.

Screenshot 2: Output 3 struk di terminal.
Pertanyaan tertulis: Kenapa Strategy Pattern berguna? Apa yang terjadi kalau semua jenis diskon di-hardcode di dalam class Kasir tanpa pakai interface?
